Spring Boot 依赖与配置文件分离打包

前言

在日常的 Spring Boot 项目开发中，我们通常会使用 spring-boot-maven-plugin 将项目打包成一个可执行的 fat jar。这个 fat jar 包含了所有的依赖，可以直接通过 java -jar 命令运行。

然而，随着项目依赖的增多，fat jar 的体积会变得越来越大（动辄几十 MB 甚至上百 MB）。这会带来几个问题：

  • 传输效率低：每次部署都需要上传整个 jar 包，在网络环境不佳时尤为痛苦。
  • 构建速度慢：每次打包都需要重新将所有依赖写入 jar 文件。
  • 依赖冗余：大多数依赖在版本迭代中并不会变化，重复打包是一种浪费。
  • 配置固化：配置文件被打进 jar，修改配置需要重新打包，不够灵活。

那么，有没有办法将依赖、配置文件和业务代码全部分离呢？答案是肯定的。本文将结合一个真实的多模块项目，详细讲解如何实现依赖与配置文件分离打包。

传统打包 vs 分离依赖打包

传统打包方式

默认情况下，Spring Boot 项目使用 spring-boot-maven-plugin 打包后，会生成一个包含所有依赖的 fat jar。项目所有依赖都会被嵌入到 jar 内部的 BOOT-INF/lib/ 目录中。

分离依赖打包

分离依赖的核心思想是：

  1. 将项目的所有依赖复制到一个独立的目录（如 lib/）。
  2. 打包时不将依赖打入 jar，只保留业务代码。
  3. 运行时通过 -Dloader.path 指定外部依赖目录。

这样做的好处显而易见：业务代码和依赖解耦，部署时只需传输几 MB 的业务 jar，依赖目录可以缓存复用。

分离打包的完整方案

我们的目标是实现三分离：

  1. 依赖分离：所有 jar 依赖放在 lib/ 目录
  2. 配置分离：所有配置文件放在 conf/ 目录
  3. 业务代码独立：jar 只包含编译后的业务代码

方式一：Mapper XML 保留在 jar 内部（推荐）

运行应用

由于依赖和配置文件都被分离出来了，我们需要同时指定它们的加载路径：

  • **-Dloader.path=lib/,conf/**：指定外部依赖和配置文件的加载路径，多个路径用逗号分隔。
  • **-jar**：指定主 jar 包。

注意：路径可以是相对路径或绝对路径。使用相对路径时，需要确保运行时的工作目录正确。

Spring Boot 配置加载顺序

Spring Boot 默认会按以下顺序加载配置文件：

  1. file:./config/（当前目录下的 config 文件夹）
  2. file:./（当前目录）
  3. classpath:/config/（jar 内的 config 目录）
  4. classpath:/（jar 根目录）

通过 -Dloader.path=conf/，我们将 conf/ 目录加入了 classpath，Spring Boot 就能正确加载外部配置文件了。

注意事项

  1. 路径问题：-Dloader.path 指定的路径可以是相对路径或绝对路径，但需要注意运行时的工作目录。
  2. 依赖更新：如果项目依赖发生变化（新增、升级），需要同步更新 lib/ 目录。
  3. 配置文件修改：修改 conf/ 下的配置文件后，需要重启应用才能生效。
  4. 多模块依赖：本项目中 admin 依赖 utils 模块，maven-dependency-plugin 会自动处理模块间依赖关系。
  5. Maven 资源过滤：maven-resources-plugin 的 filtering=true 会替换 ${} 占位符，注意避免与 Spring Boot 的占位符冲突。
  6. Mapper XML 文件：建议保留在 jar 内部，避免外部化带来的安全问题。

总结

分离打包是一种优化 Spring Boot 部署效率的有效手段。通过 spring-boot-maven-plugin、maven-dependency-plugin、maven-resources-plugin 和 maven-jar-plugin 的配合，我们可以实现：

  • ✅ 大幅减小业务 jar 体积（从几十 MB 降至几 MB）
  • ✅ 依赖目录可缓存复用，提升部署速度
  • ✅ 配置文件外部化，修改配置无需重新打包
  • ✅ 业务代码与依赖、配置解耦，便于版本管理和多环境部署

核心插件一览

插件作用
spring-boot-maven-plugin打包业务代码，排除依赖（layout=ZIP）
maven-jar-plugin排除配置文件不打进 jar
maven-dependency-plugin复制依赖到 lib/ 目录
maven-resources-plugin复制配置文件到 conf/ 目录
maven-surefire-plugin打包时跳过测试

如果你的项目也经常面临大 jar 包传输或修改配置需重新打包的困扰，不妨试试这种分离打包方式。

本文基于 Spring Boot 2.7.3 + Maven 编写，配置思路适用于大多数 Spring Boot 2.x/3.x 项目。
