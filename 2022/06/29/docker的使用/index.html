常用命令

容器相关

  • docker

    • ps

      • 查看所有容器

        • -a

          • 启动已停止的容器

        • -l

          • 查看最后一次创建的容器

    • run image

      • 启动容器运行镜像

        • -t

          • 在新容器指定一个伪终端或终端

        • -i

          • 允许对容器内进行输入

        • -d

          • 后台运行，以进程模式启动

        • -P

          • 映射端口到物理机上

            • 自动映射

        • -p

          • 映射端口到物理机上

            • 指定端口

              • real port : docker port

    • start containerId

      • 启动一个已停止的容器

    • stop containerId

      • 停止一个容器

    • restart containerId

    • attach containerId

      • 以后台模式运行的容器 执行某条命令

        • 例如 docker attach container Id /bin/bash

          • 退出bash 时容器会停止运行

    • exec containerId

      • 以后台模式运行的容器 执行某条命令

        • 例如 docker exec container Id /bin/bash

          • 退出bash 时不会停止容器

    • export containerId

      • 导出容器到本地文件

    • import filePath

      • 导入容器 cat docker/ubutu.tar | docker import - test/ubuntu:v1

        • v1 是tag内容

    • rm -f containerId

      • 删除容器

        • 删除所有停止的容器

          • docker container prune

    • logs

      • 查看容器内部日志

      • -f

        • 让 docker logs 像使用 tail -f 一样来输出容器内部的标准输出。

    • top container

      • 查看内部进程

Dockerfile文件

  • FROM

    • base镜像

      • 基于某个镜像构建

  • 运行命令

    • RUN

      • 构建时运行

        • 每一条RUN指令会多加一层镜像，造成镜像过大，可用&&连接多条命令

      • 格式

        • RUN 命令
        • RUN [‘可执行文件’,’参数1’,’参数2’]

    • CMD

      • 容器启动是运行

        • 注意

          • 只有最后一条CMD指令会被执行

      • 格式

        • CMD 命令

        • CMD [‘可执行文件’,’参数1’,’参数2’]

        • CMD [‘参数1’,’参数2’]

          • 该写法为entry point 指令指定的程序提供默认参数

            • 入口点

    • ENTRYPOINT

      • 类似CMD指令,可与docker run 配合使用可变参数，与CMD配合使用默认参数

        • 同CMD一样，仅最后一个生效

      • 格式

        • ENTRYPOINT [‘nginx’,’-c;]CMD [“/path”]

          • docker run nginx:test

            • 不指定参数，nginx的-c参数值为/path

          • docker run nginx:test -c ‘/otherPath’

            • 指定参数 nginx的-c参数为/otherPath

  • ENV

    • 设置环境变量，后续的指令可以使用这个变量

      • 访问使用$变量名

    • 格式

      • ENV key value
      • ENV key1=value1 key2=value2

  • ARG

    • 与ENV作用一只，只是作用域不同，ARG只有build能访问到
    • 可在 docker build 时通过 –build-arg 参数名=值来覆盖

  • VOLUME

    • 定义匿名数据卷，在启动容器时忘记挂载数据卷，会自动挂载到匿名卷

    • 格式

      • VOLUME [path1,path2]
      • VOLUME path

    • docker run -v 指定挂载点

  • EXPOSE

    • 仅声明端口

  • WORKDIR

    • 工作目录

  • COPY

    • 复制文件

  • ONBUILD

    • 延迟构建命令的执行

      • 这次构建的镜像被新的dockerfile引用且构建时运行

  • LABEL

    • 元数据

      • LABEL key=value key2=value2

  • USER

    • 改变执行用户

  • HEALTHCHECK

    • 指定某个程序或指令来监控容器服务的状态
