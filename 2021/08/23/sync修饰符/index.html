.sync修饰符

在 Vue 开发中，组件之间的通信是一个核心话题。通常，父组件通过 props 向子组件传递数据，子组件通过 $emit 触发事件来通知父组件进行修改。对于“父子组件间某个数据需要双向绑定”的场景，Vue 提供了 .sync 修饰符，作为一种语法糖，简化了代码。

使用 .sync 的注意事项

  1. 不能与表达式一起使用：:title.sync="父组件数据" 绑定的必须是一个数据属性名，不能是像 :title.sync="user.name + '!'" 这样的表达式。

  2. 支持多个 .sync 绑定：一个组件上可以同时使用多个 .sync 修饰符，互不干扰。

  3. 支持 v-bind 对象绑定：如果需要传递多个 prop 并都使用 .sync，可以使用 v-bind.sync="object"。但需要注意，v-bind.sync 在 Vue 2.3 引入，在 Vue 2 的后续版本中行为可能有所调整（官方更推荐显式列出每个 .sync 绑定，以提高可维护性）。

  4. 子组件中不要直接修改 prop：.sync 仅仅是语法糖，并没有改变单向数据流的原则。子组件中绝对不能直接修改 prop（例如 this.title = '新标题'），这样做 Vue 会在控制台发出警告。必须通过 $emit('update:title', newValue) 来请求父组件更新。

与 v-model 的对比

  • v-model 通常用于在单个组件上实现值的双向绑定，默认绑定 value prop 和 input 事件。
  • .sync 更灵活，可以给任意多个 prop 显式指定“更新事件”，事件名称格式为 update:propName。

在 Vue 3 中，v-model 得到了增强，可以指定多个绑定，例如 v-model:title="title"，这实际上融合了 .sync 的使用场景。但在 Vue 2 项目中，.sync 依然是处理多个 prop 双向绑定的利器。

总结

  • .sync 是一个语法糖，简化了父组件监听子组件更新特定 prop 事件的代码。
  • 子组件通过 $emit('update:propName', newValue) 触发更新。
  • 保持了单向数据流的原则（子组件不直接修改 prop）。
  • 适合在需要“双向绑定”多个 prop 的场景下使用，让代码更加简洁优雅。
